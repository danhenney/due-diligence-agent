# 비평 분석 (Critique Analysis)

## 목적
전체 DD 패키지의 품질을 5가지 기준으로 채점하고, 점수가 낮은 항목에 대해 **구체적이고 실행 가능한 개선 피드백**을 제공한다.

---

## Step 1. 채점 기준 (각 1~10점)

### 1. 논리성 (LOGIC)
- 결론이 근거에서 논리적으로 도출되는가?
- 인과 관계가 타당한가?
- 가정이 명확히 기술되고 합리적인가?
- 10 = 완벽한 논리 체계 | 5 = 일부 논리 빈틈 | 1 = 모순적 추론

### 2. 완성도 (COMPLETENESS)
- Phase 1의 6가지 차원이 모두 충분히 다루어졌는가?
- 데이터 또는 분석에 중대한 공백이 있는가?
- 밸류에이션이 충실한가 (DCF + 비교법 + 자산기반)?
- 10 = 종합적, 빈틈 없음 | 5 = 주목할 공백 존재 | 1 = 주요 섹션 누락

### 3. 정확도 (ACCURACY)
- 사실과 수치가 정확하고 최신인가?
- 리뷰 에이전트가 모순이나 오래된 데이터를 발견했는가?
- 재무 수치가 라이브 소스에서 가져온 것인가?
- 시장 규모 추정이 출처와 함께 합리적인가?
- 10 = 전부 검증, 최신 | 5 = 일부 미검증 주장 | 1 = 중대한 오류

### 4. 서술 편향 (NARRATIVE BIAS)
- 분석이 균형 잡히고 객관적인가?
- 과도한 낙관 또는 비관을 피하고 있는가?
- 리스크가 기회 대비 적절한 비중을 받고 있는가?
- 추천이 편향이 아닌 근거에 의해 정당화되는가?
- 10 = 완벽한 균형 | 5 = 눈에 띄는 편향 | 1 = 심각한 편향

### 5. 인사이트 실효성 (INSIGHT EFFECTIVENESS)
- 투자 논거가 참신하고 차별화되었는가?
- 투자위원회가 의사결정에 유용하다고 판단할 것인가?
- 표면적 분석을 넘어서는가?
- 10 = 탁월한 인사이트 | 5 = 적절하나 뻔함 | 1 = 인사이트 가치 없음

---

## Step 2. 채점 규칙

- 공정하고 보정된 채점을 한다. 라이브 데이터를 활용한 유능한 자동 DD 분석은 6~8점이 적절하다.
- 1~4점: 실제로 분석 품질이 떨어지는 경우에만 (데이터 누락, 잘못된 결론)
- 9~10점: 인간 수준의 예외적 품질에만
- 총점(5개 기준 합계) 범위: 5~50점
- **중요**: 자동 분석의 본질적 한계(예: 경영진 인터뷰 불가)로 감점하지 않는다. 가용한 데이터 기준으로 채점한다.

💡 팁: 총점 35+ AND 모든 기준 7+ → 통과, 미달 시 피드백 루프가 작동한다. 불필요하게 후한 점수는 품질 저하를 초래한다.

---

## Step 3. 개선 피드백 작성

7점 미만인 각 기준에 대해:
- 무엇이 개선되어야 하는지 **구체적으로** 기술
- 어떤 에이전트가 재실행되어야 하는지 명시
- 해당 에이전트가 무엇을 수정해야 하는지 기술

💡 팁: "더 조사 필요"가 아니라 "financial_analysis 에이전트가 DCF에서 WACC 근거를 누락했으므로 재실행 필요"처럼 구체적으로 쓴다.

---

## 전체 규칙

1. 자동 분석의 본질적 한계로 감점하지 않음
2. 7점 미만 기준에는 반드시 weak_agents와 specific_improvements 명시
3. 피드백은 구체적이고 실행 가능해야 함 — 막연한 지적 금지
4. 총점은 5개 기준의 정확한 합계

---

## JSON Output Schema

Return a JSON object with this exact structure:
{
  "summary": "<2-3 sentence overall assessment>",
  "logic": 0,
  "completeness": 0,
  "accuracy": 0,
  "narrative_bias": 0,
  "insight_effectiveness": 0,
  "total_score": 0,
  "feedback": [
    {
      "criterion": "...",
      "score": 0,
      "assessment": "...",
      "weak_agents": ["<agent_name that needs improvement>"],
      "specific_improvements": ["..."]
    }
  ]
}

## 점수 캘리브레이션 가이드라인

점수 인플레이션 방지를 위한 캘리브레이션:

| 점수 | 의미 | 빈도 기대치 |
|------|------|-----------|
| 9-10 | 탁월 — 추가 검증 불필요 | 전체의 ~5% |
| 7-8 | 양호 — 사소한 보완 필요 | 전체의 ~40% |
| 5-6 | 보통 — 상당한 보완 필요 | 전체의 ~35% |
| 3-4 | 미흡 — 재작성 권고 | 전체의 ~15% |
| 1-2 | 부적합 — 전면 재작성 | 전체의 ~5% |

**평균 점수가 8점 이상이면 자체 점검**: 정말 그만큼 우수한가, 기준이 느슨한 건 아닌가.

## 이전 DD 대비 상대 평가

히스토리 데이터가 있으면 이전 DD 보고서 대비 상대 비교:
- "이전 3건의 DD 평균 점수: X점. 이번 점수: Y점"
- 특정 축이 이전 대비 현저히 높거나 낮으면 그 이유 설명
- 히스토리 없으면 이 섹션 생략

## 개선 우선순위 매트릭스

7점 미만 항목에 대해:

| 항목 | 현재 점수 | 개선 난이도 | 개선 영향도 | 우선순위 |
|------|---------|-----------|-----------|---------|
| ... | ... | 상/중/하 | 상/중/하 | P1/P2/P3 |

영향도 높고 난이도 낮은 것 = P1 (즉시 개선).


## 과잉 생성 방지 가드레일 (MANDATORY)

- 새로 추가된 프레임워크(TRL, ESG, Conviction, Expected Loss 등)는 **데이터가 확보된 항목만** 작성.
- 데이터 없는 셀은 `N/A` + "미확보 사유" 한 줄. **추정치로 채우지 말 것.**
- 출력 크기: 3KB~15KB. 3KB 미만이면 분석 부족, 15KB 초과이면 군더더기 의심.
- 기존 Step 1~N은 필수. 신규 프레임워크 테이블은 데이터 확보 시에만.
- 테이블 채우기에 tool call을 추가로 소비하지 말 것 — 기존 검색 결과 내에서만 채울 것.
